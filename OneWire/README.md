# BTT Pi OneWire Overlay
Подключение датчика DS18B20

В образе уже имеется модуль ядра w1-gpio, но он не принимает параметров и не знает на каком пине висит датчик. Обычно это решается загрузкой оверлея для дерева устройств, но производитель его не поставил по какой-то причине. Исправим данное недоразумение.

> [!CAUTION]
> Все что вы делаете, вы делаете на свой страх и риск, автор доработок не несет ответственности за любые повреждения, которые могут случиться из-за использования данных модификаций.

> [!IMPORTANT]
> Перед любыми манипуляциями желательно сделать резевные копии всех изменяемых файлов.

## 1. Загрузка оверлея
Скачать файл un50i-h616-w1-gpio.dtbo и разместить его по пути `/boot/dtb/allwinner/overlay/`. Можно использовать Mobatxertm, WinSCP или достать SD-карту и подключить ее к компу (тогда путь будет `БУКВА_КАРТЫ:\dtb\allwinner\overlay\`).

## 2. Настройка оверлея
Для включения оверлея необходимо отредактировать файл `armbianEnv.txt`, добавив код

```
overlays=w1-gpio
#param_w1_pin=PC7
#param_w1_pin_int_pullup=0
```

 По умолчанию система будет искать сигнал датчика на пине PC7, если пин отличается, то необходимо раскоментировать строчку param_w1_pin и изменить наименование пина.

## 3. Настройка klipper

Документация https://www.klipper3d.org/Config_Reference.html#ds18b20-temperature-sensor

Найдем серийный номер датчика

```shell
sudo ls /sys/bus/w1/devices/
```

Должен появиться примерно такой вывод

```
28-011319f769fe  w1_bus_master1
```

Добавим секцию в printer.cfg

```ini
[temperature_sensor my_sensor]
sensor_type: DS18B20
serial_no: 28-011319f769fe
sensor_mcu: CB1
```

_Значение `sensor_mcu` должно указывать на host\_mcu, обычно это `CB1` по умолчанию._

После рестарта клиппера, новый датчик должен появиться в Mainsail/Fluidd автоматически.

> [!TIP]
> При необходимости можно пересобрать dtbo из dts `dtc -@ -I dts -O dtb -o sun50i-h616-w1-gpio.dtbo sun50i-h616-w1-gpio.dts`
